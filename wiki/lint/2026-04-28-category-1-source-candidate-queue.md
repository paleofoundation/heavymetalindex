# Category 1 Source Candidate Queue

This queue is generated from the current triage manifest at `/Users/karenpendergrass/Documents/heavy-metal-index/triage-output/current/manifest.csv`. It is a discovery aid only. Candidate ranking is based on deterministic manifest signals and filename/path text; it is not evidence and must not be cited in wiki prose.

Before any product-category page can move text out of `<!-- UNCITED -->` status, a candidate source must be read and promoted into `wiki/sources/<cite-key>.md`.

## Scoring Signals

- Matrix hits for infant formula, infant food, rice, cereals, root vegetables, seafood, and fruit juice.
- Metal hits mapped to the row metal targets. Triage uses coarse `As` and `Hg`, so iAs/tAs and MeHg/tHg require manual speciation verification.
- Regulatory-value flags, percentile flags, LOQ/LOD flags, agency/report flags, jurisdiction flags, first-page DOI, and review/meta-analysis signals.
- Row-specific terms in filename/path when available.
- Negative flags for likely wrong-side splits, such as rice showing up in non-rice rows.

## Immediate Caveats

- Many PDFs are named only by `FM_<id>.pdf`, so row assignment can be under-specific until the PDF is opened.
- Formula powder vs RTF and soy vs non-soy cannot be trusted from manifest signals alone.
- Puree, mixed-meal, and snack rows need manual confirmation because the current triage rule set does not encode all Category 1 row distinctions.
